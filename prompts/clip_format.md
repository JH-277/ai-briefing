# 숏츠형 AI 브리핑 — 작성 가이드

## 목표

- **3분 숏츠**처럼: 훅 → 공감 → 설명 → **업무 사례** → **오늘 할 한 가지**
- 독자가 **“아, 그래서 회의에서 그 말이…”** 하고 연결되게
- **메타 문구 금지** (“클립 1”, “왜 내 얘기” 같은 **설계 라벨**을 본문에 노출하지 않음)

## 본문 구조 (5블록, 제목은 자연스럽게)

```markdown
# [제목 — 궁금증/상황형]

## 첫 3초
(한 줄 질문 또는 반전. “○○만 들었는데?”)

## 이런 적 있지 않나
(2~4문장, 2인칭 ‘당신’ 또는 ‘우리’. 회의·메일·출근 장면)

## 쉽게 말하면
(비유 1개 + 풀어쓴 설명 3~5문장. 전문어는 바로 풀어쓰기)

## 회사에서 이렇게 쓰인다
(구체 사례 1~2개: 팀, 상황, Before→After)

## 오늘 해볼 것
(딱 1가지, 1~2문장. 월요일 아침에도 할 수 있는 수준)

## 카카오 요약
(아래 규칙)
```

## 톤

- **친구가 카톡으로 설명** (존댓말, 딱딱한 보고서 X)
- **한 블록 = 3~5문장** 넘기지 않기
- 전체 **500~700자** (카카오 제외)
- ❌ 이모지 라벨(🟢🟡🔵) 남발, ❌ 표·코드, ❌ 영어 약어만 던지기

## 카카오 요약

- **250~450자**, **문단 2~3개** (숏츠 자막처럼 읽히게)
- 구조: **훅 1문장 → 핵심 2~3문장 → 오늘 할 것 1문장**
- `#`, `##` 없이 평문

## 주제 선정 (topics.json)

- 고정 syllabus X → **회의·뉴스·업무에서 실제로 들리는 말** 중심
- 제목은 **상황형** (“~할 때”, “~만 오갔을 때”)
- 후보는 `data/topic_backlog.json`에 쌓고, 확정 시 library md 작성 후 topics에 추가
- 10일 1사이클 돌면 backlog에서 **2~3개 교체**하며 신선도 유지
