# 비개발자가 Cursor를 쓰는 이유

## 첫 3초

Cursor = **개발자 전용?**  
아니요. **“말로 일 시키는 AI + 자동화”** 쪽에 가깝습니다.

## 이런 적 있지 않나

엑셀 정리, **반복 메일**, **자료 조사**, **카톡으로 받을 브리핑**…  
“**이거 매일** 해야 하는데.”  
코드는 못 짜도 **“이렇게 해줘”**는 할 수 있죠.

## 쉽게 말하면

Cursor는 **AI 채팅 + 내 PC 파일 + 스크립트 실행**이 한곳에 있는 도구예요.  
**코딩**도 되지만, 비개발자는 **조사·정리·연결**에 씁니다.  
예: “**이 개념 쉽게 정리해서 카톡으로 보내**”, “**매일 7시 반** 이거 실행” — **지금 이 프로젝트**가 그 예시.

## 회사에서 이렇게 쓰인다

**기획:** 회의록 → **임원용 3줄** + **액션 아이템** 추출  
**마케팅:** 경쟁사 URL → **비교표 초안** (숫자는 사람이 검증)  
**총무/HR:** FAQ 초안 → **직원용 쉬운 말**로 다시 쓰기  
**본인:** AI 개념 **숏츠 브리핑** → **카카오 자동 전송** (GitHub 연동)

개발팀 없이 **“반복·정리·전달”** 업무를 줄일 때 특히 잘 맞아요.

## 오늘 해볼 것

Cursor에 **업무 하나**만 던져 보세요.  
“**내일 회의 agenda** 초안, **참석자 4명**, **30분**, **불릿만**.”  
— **코드** 말고 **결과물**을 달라고 하면 됩니다.

## 카카오 요약

Cursor는 개발자만 쓰는 게 아니에요. AI 채팅 + 내 파일 + 자동 실행이 같이 있어서, **조사·정리·반복 업무**에 씁니다. “매일 카톡으로 브리핑” 같은 것도 가능해요.

실무: 회의록→임원용 3줄, FAQ→쉬운 말로 재작성, 경쟁사 URL→비교 초안(숫자는 직접 확인).

오늘 **코드 말고 결과물** 하나만 시켜 보세요. “내일 회의 agenda, 불릿만, 30분.”
