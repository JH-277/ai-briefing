# API·MCP, 뉴스만 보면 헷갈릴 때

## 첫 3초

“**API 오픈**”, “**MCP 표준**” — **뉴스 헤드라인**만 보면 **같은 말** 같죠?

## 이런 적 있지 않나

“우리도 **API 연동**?”  
“**MCP**로 **Slack** 붙인다?”  
**IT**와 **기획**이 **다른 단어** 쓰는데 **회의록**엔 **둘 다** 적힘.

## 쉽게 말하면

**API** = 가게 **주문 창구**.  
메뉴·방법이 **정해져** 있어요. “**날씨 API**”면 **날씨만** 주문 가능.  

**MCP** = AI한테 **여러 가게 카드**를 **한 지갑**에 넣어 주는 **규격**.  
Cursor가 **Slack·파일·GitHub**를 **같은 방식**으로 쓰게 하는 게 MCP 쪽.  

**MCP 안에서** 실제로는 **각 서비스 API**를 부릅니다. **대체**가 아니라 **AI용 포장**.

## 회사에서 이렇게 쓰인다

**API 연동:**  
CRM ↔ **주문 시스템** — “**주문 넣기**” 창구 하나 파는 **프로젝트** (개발·외주)  

**MCP:**  
**Cursor**로 “**이 PR 요약**”, “**슬랙에 알림**” — **AI 비서**가 **도구**를 고르게  

**기획자**는 API = **시스템끼리 연결**, MCP = **AI가 쓰는 연결** 정도만 알아도 **회의 번역**이 됩니다.

## 오늘 해볼 것

뉴스·회의에서 단어 들으면 **한 번만** 구분:  
**“사람/시스템용 창구(API)?”** vs **“AI용 도구 연결(MCP)?”**

## 카카오 요약

API는 **정해진 주문 창구**(날씨 API면 날씨만). MCP는 **AI가 Slack·파일 등을 같은 방식으로 쓰게 하는 연결 규격**이에요. MCP가 API를 없애는 게 아니라, AI가 쓰기 쉽게 묶어 주는 층이에요.

회사: CRM↔주문 연결 = API 프로젝트. Cursor로 PR 요약·슬랙 알림 = MCP 느낌.

오늘: **창구(API) vs AI 연결(MCP)** 한 번만 구분해 보세요.
