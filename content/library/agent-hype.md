# AI Agent, 지금 당장 필요해?

## 첫 3초

“에이전트 도입” — 챗봇이면 충분한데 비싼 프로젝트가 될 수 있어요.

## 이런 적 있지 않나

“AI Agent로 영업 자동화!”  
벤더 데모는 멀티스텝·도구 연동까지 보여줘서 멋있어요.  
근데 우리는 FAQ도 아직 정리 안 된 상태.

## 쉽게 말하면

챗봇은 물으면 답하는 **안내 데스크**예요.  
에이전트는 “이번 주 리포트 만들고 팀에 보내줘”처럼 **여러 단계**를 스스로 밟는 **비서**에 가깝습니다.  

문서·권한·검증이 갖춰진 뒤에야 에이전트 효과가 커요.  
FAQ 없이 에이전트부터 = 비싼 데모만 보는 경우가 많습니다.

## 회사에서 이렇게 쓰인다

**Agent가 과한 경우:** FAQ 50개도 없고, “알아서 해”만 있고 성공 기준이 없을 때  

**Agent가 맞는 경우:** 매주 같은 리포트·발송·티켓 분류가 있고, 사내 문서(RAG)와 승인 도구가 이미 있을 때  

Before: “올해 Agent 도입” (범위 모호)  
After: “3분기 FAQ 봇 → 4분기 반복 업무 1개만 에이전트로”

## 오늘 해볼 것

회의에서 Agent 나오면 한 가지만 물어보세요.  
“지금 필요한 건 **한 번에 끝나는 질문**인가요, **여러 단계 업무**인가요?”  
전자면 챗봇부터면 됩니다.

## 카카오 요약

Agent는 챗봇보다 한 단계 위예요. 목표를 주면 검색·발송·수정처럼 여러 단계를 스스로 하는 비서에 가깝죠. FAQ·문서·권한 없이 Agent부터 하면 비싼 데모가 되기 쉬워요.

실무: FAQ도 없는데 Agent → 과함. 반복 리포트·티켓 분류 + 사내 문서 연결 → Agent 검토할 만함. “올해 Agent” 대신 “3분기 FAQ, 4분기 반복 업무 1개”처럼 순서를 적어 보세요.

오늘: 한 번에 끝나는 질문이면 챗봇부터, 여러 단계면 Agent를 검토.
